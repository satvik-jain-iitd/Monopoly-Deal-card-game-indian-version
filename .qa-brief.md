## Test Cases Per Bug

### Bug 1: Online Multiplayer
- **Test A:** Host creates room → verify lobby appears with "waiting for players"
- **Test B:** Guest joins via WebSocket → verify both see each other in lobby
- **Test C:** Host starts game → verify game state propagates to guest
- **Test D:** Guest plays a card → verify host receives and processes GAME_ACTION
- **Edge case:** Guest disconnects mid-game → verify host detects PLAYER_LEFT
- **Edge case:** Slow network (simulated via DevTools throttling) → verify ICE completes within timeout

### Bug 2: Wild Card Colour Lock
- **Test A:** Play a two-colour wild (e.g., Pink/Orange) as Pink → verify Pink set shows it
- **Test B:** On next turn, tap the wild → verify colour toggle UI appears
- **Test C:** Flip to Orange → verify it moves to Orange set
- **Edge case:** Full wild → verify can change to ANY colour including Railroad/Utility
- **Edge case:** Changing wild colour breaks a set → verify buildings on that set deactivate

### Bug 3: JSN for Deal Breaker
- **Test A:** Target has JSN in hand → play Deal Breaker → verify JSN prompt shows
- **Test B:** Target plays JSN → Deal Breaker is cancelled → verify no steal occurs
- **Test C:** Original actor has JSN → plays counter-JSN → verify steal proceeds
- **Edge case:** Target has NO JSN → verify Deal Breaker completes immediately
- **Edge case:** Target has Insurance → verify Insurance blocks Deal Breaker WITHOUT JSN prompt (current behaviour correct)

### Bug 4: Rent Payment – Hand Visibility
- **Test A:** Player owes rent → verify payment sheet shows Haath section
- **Test B:** Hand section shows correct card count and names
- **Edge case:** Empty hand → verify "Haath khaali" message

### Bug 5: Rent Card with Zero Properties
- **Test A:** Player has NO green/blue properties → try to play Green+Blue Rent → verify button is greyed out
- **Test B:** Player has at least one green property → verify Green+Blue Rent can be played
- **Test C:** Player banks the rent card instead → verify normal banking works
- **Edge case:** All properties are wild-assigned to non-matching colours → verify Rent can't be played

## Rejection Criteria
- Any test fails → reject + document exact steps to reproduce
- Lint/build breaks → reject
- Online multiplayer mode breaks existing pass-and-play mode → reject
- Wild card toggle accidentally changes non-wild property colour → reject
