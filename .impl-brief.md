## Bug 1: Online Multiplayer Communication Failure

**Root Cause:** `useWebRTC.js` line 4 — `RTC_CONFIG = { iceServers: [] }`. No STUN/TURN. Peers behind NAT can't connect.

**Fix:**
1. Add at least Google's public STUN server: `iceServers: [{ urls: 'stun:stun.l.google.com:19302' }]`
2. In `useWebRTC.js` `createOffer` / `createAnswer`: add timeout fallback for `waitForICE` (currently 3s, might be too short on slow networks)
3. In `useMultiplayer.js`: ensure guest's HELLO is sent reliably (retry if `send()` fails because WS isn't open yet)
4. In `rtcUtils.js` `stripSDP`: filter CANDIDATE lines more carefully — currently only `host` candidates are kept; need to also keep `srflx` candidates so LAN IP detection works across WiFi/hotspot
5. In `App.jsx` `handleRoomReady`: host sends ROSTER explicitly; verify this propagates

## Bug 2: Wild Card Colour Lock (Pass & Play)

**Root Cause:** `gameLogic.js` `playPropertyCard` sets `card.color = targetColor` permanently. No mechanism exists to change wild card colour later.

**Fix:**
1. Mark wild property cards in the property area with an `isWild: true` flag so UI can identify them
2. In `useGameState.js` `gameReducer`, add a new action `CHANGE_WILD_COLOR` that toggles the colour of a placed wild card
3. In `GameScreen.jsx`, during PLAY phase, render a small "🔄" button on wild property cards in the player's board that dispatches `CHANGE_WILD_COLOR`
4. For online multiplayer, broadcast `CHANGE_WILD_COLOR` as a `GAME_ACTION`

## Bug 3: Missing JSN Chain for Deal Breaker

**Root Cause:** `useGameState.js` `DEAL_BREAKER_STEAL` case (line 419) executes steal immediately, never pauses for JSN.

**Fix:**
1. Before executing deal breaker steal, check if victim has JSN in hand
2. If yes, add a JSN_RESPONSE phase pause (same pattern as Rent JSN)
3. Wrap the steal logic inside a sub-phase: `DEAL_BREAKER_STEAL` only executes after JSN check passes
4. In `ActionModal.jsx`, handle the JSN prompt for Deal Breaker target (show "Play Just Say No?" before the steal resolves)
5. Support the full chain: target JSN → actor counter-JSN

## Bug 4: Rent Payment – Hidden Hand Information

**Root Cause:** `ActionModal.jsx` `PaymentSheet` only shows bank + properties + buildings — NOT hand cards.

**Fix:**
1. Add payer's hand cards to the `PaymentSheet` in a "✋ Haath" section
2. Arrange sections: Cash (left) → Actions/Rent (centre) → Hand properties (centre-right) → Board properties (right)
3. Hand cards should be read-only (can't be used for payment directly) but visible so player can make informed decisions

## Bug 5: Playing Rent Card with Zero Matching Properties

**Root Cause:** `GameScreen.jsx` `PlayOptions` shows "Rent Maango" button for ANY rent card regardless of player's property area.

**Fix:**
1. In `GameScreen.jsx` `PlayOptions`, check if the current player owns at least one property matching the rent card's colour set
2. If not, grey out the "Rent Maango" button and show tooltip "Zaroorat: at least one {colour} property"
3. In `RentSelector`, add early guard: if `eligibleColors.length === 0`, show message and cancel button
4. In `useGameState.js` `PLAY_RENT` case, add engine-level guard (defense in depth)
