## Sprint Goal

Fix 5 bugs in Dhandha (Monopoly Deal Indian version) so the game is fully playable online and offline, with correct rules enforcement.

---

## Scope — WHAT We Build

### Bug 1 — Online Multiplayer Communication Fix

**What:**
- Fix WebSocket lobby handshake race condition
- Add WebSocket connection status + error display in LobbyScreen
- Add HELLO send only after WS is OPEN (event-driven, not timeout-based)
- Improve ICE candidate handling in rtcUtils.js (keep srflx candidates too)
- Add STUN server to RTC_CONFIG
- Add reconnection logic for dropped WebSocket

**Acceptance Criteria:**
1. Host creates room → sees "Room created, waiting for players..." with connection ✅ indicator
2. Guest joins room → host sees guest appear in lobby (ROSTER updates)
3. Host starts game → BOTH players see game screen with correct state
4. Guest disconnects → host sees "Player left" notification within 5s
5. Guest reconnects → lobby works again (not required to restore mid-game)
6. Connection failure → error message visible in lobby (not just spinner)

### Bug 2 — Wild Card Colour Lock

**What:**
- Don't mutate `card.color` when playing wild property
- Use `card.assignedColor` field for tracking current colour assignment
- Add `CHANGE_WILD_COLOR` action in the reducer
- Add tap-to-toggle UI on owned wild cards during player's turn
- For online: broadcast colour change as GAME_ACTION

**Acceptance Criteria:**
1. Play dual-colour wild (e.g., Pink/Orange) → assigned colour works for set completion
2. On current player's turn, tap wild card on board → colour picker appears
3. Change colour → card moves to new colour set ✅
4. Full wild → can be assigned to ANY colour (including Railroad/Utility)
5. Changing wild that breaks a set → buildings on broken set deactivate
6. Non-current player cannot change wild colour
7. Pass-and-play: colour change survives device pass (hand-off)

### Bug 3 — JSN Chain for Deal Breaker

**What:**
- After DEAL_BREAKER_SELECT (player picks target), don't execute steal immediately
- Route through ACTION_RESPONSE phase to prompt victim for JSN
- Support JSN counter-chain (original actor can counter victim's JSN)
- Insurance pre-block remains (Insurance consumed, no JSN prompt needed)

**Acceptance Criteria:**
1. Play Deal Breaker on a complete set → victim sees "Play Just Say No?" prompt
2. Victim plays JSN → Deal Breaker cancelled, JSN goes to discard
3. Original actor has JSN → can counter-victim's JSN, steal proceeds
4. Victim has NO JSN → Deal Breaker executes normally after short delay
5. Victim has Insurance → Insurance blocks Deal Breaker silently (existing behaviour preserved)
6. JSN chain depth = 1 (one counter only, per original game rules)

### Bug 4 — Rent Payment Hand Visibility

**What:**
- Add payer's hand cards to PaymentSheet assets
- Show "✋ Haath" section with non-property hand cards (money + action/rent)
- Hand cards are read-only display (not selectable for payment unless rules allow paying from hand)

**Acceptance Criteria:**
1. Rent payment modal shows "✋ Haath" section with hand cards
2. Hand cards show correct names and values (read-only)
3. Player can see bank + properties + hand simultaneously
4. Empty hand shows "✋ Haath khaali" message
5. Layout consistent with main game (left → centre → right ordering)

### Bug 5 — Rent Card with Zero Matching Properties

**What:**
- PlayOptions: only show "Rent Maango" when player has ≥1 matching property
- RentSelector: guard empty eligibleColors with message
- PLAY_RENT reducer: return state if playerProps.length === 0
- Don't consume doubleRentActive flag when rent fails

**Acceptance Criteria:**
1. Player has zero Red/Yellow properties → Red/Yellow Rent card "Rent Maango" button greyed out
2. Player has one Red property → Red/Yellow Rent card playable
3. Wild Rent → playable if player has at least one property of ANY colour
4. Player banks the rent card instead → normal banking works
5. Reducer guard returns state unchanged (no consumed play, no consumed doubleRent)
